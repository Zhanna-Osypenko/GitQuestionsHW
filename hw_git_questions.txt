1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

Чтобы отказаться от изменений в файле (файлах) в рабочем каталоге, вы можете использовать команду "git checkout <file>"/"git checkout ."

2. Как вернуть файл из индекса в рабочий каталог?

Командой git restore --staged <file>. Эта команда удаляет файл из индекса, сохраняя его состояние в рабочем каталоге без изменений.

3. Как отменить ошибочный commit ?

git revert <commit>. Эта команда создаст новый коммит, который отменит изменения, внесенные указанным коммитом. Таким образом, история остается неизменной, но ошибочные изменения будут отменены.

4. Как создать новую ветку и переключится в нее?

git checkout -b <branch>. Эта команда создает новую ветку с указанным именем и автоматически переключает нас на эту ветку.

5. В чем отличие между fast-forward merge  and non fast-forward merge?

Fast-forward слияние (fast-forward merge) происходит, когда ветка, на которую мы хотим выполнить слияние, находится впереди текущей ветки, и Git может просто переместить указатель текущей ветки вперед, чтобы указывать на тот же коммит, на который указывает ветка, которую вы сливаете. Нет дополнительных коммитов слияния, и история остается линейной.

Non fast-forward слияние (non fast-forward merge) происходит, когда ветка, на которую мы хотим выполнить слияние, имеет новые коммиты, которых нет в текущей ветке. В этом случае Git создает новый коммит слияния, который объединяет изменения из обеих веток и создает новую точку слияния в истории.

6. Когда может возникнуть конфликт слияния веток, как  его разрешить?

Конфликт слияния веток может возникнуть, когда Git не может автоматически объединить изменения из разных веток из-за конфликта в измененном коде или структуре файлов. Разрешение конфликта требует ручного вмешательства. Мы должны открыть конфликтующий файл в текстовом редакторе и разрешить конфликт вручную, выбрав нужные изменения или редактируя код так, чтобы он соответствовал нашим требованиям. После разрешения конфликта нужно добавить измененные файлы в индекс (git add) и завершить слияние (git commit).

7. Можно ли удалить ветку? Как? 

Да, с помощью команды git branch -d <branch>.
